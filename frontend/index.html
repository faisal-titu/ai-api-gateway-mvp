The Architecture

From Query to Result

01

Your Input

You type "a cat sitting on a laptop" or upload an image. The frontend sends this to our FastAPI + CLIP backend.

02

Vector Embedding

The CLIP model converts your input into a 512-dimensional vector offering semantic understanding of the content.

03

k-NN Search

FAISS matches your vector against 25,000+ indexed images using cosine similarity and returns the top K results in ~3ms.
